### Ledger-charge policy ‚Äî the return-blind-screen bright line (operator-ruled 2026-07-12)
Which constructions charge the effective-N ledger is decided by **whether the step could fish for
a good result**, not by whether a return stream was persisted (a bookkeeping fact one could dodge):
- A construction **screened out on a return-blind, pre-registered criterion** ‚Äî **D8** independence,
  prefix-invariance, eligibility, data-integrity (bands fixed in advance, never sees returns) ‚Äî is
  **NOT charged.** Such a screen cannot select for a lucky pass, and effective-N exists precisely to
  price the probability of a false PASS.
- A construction that was **RUN and evaluated on returns is CHARGED, always, even if discarded** ‚Äî
  it is a real bite at performance. (Had we run V, seen it fail, then tried V_resid, that second
  bite must be charged.)
- First application: candidate #2's **V / V-A were D8-screened out and are NOT charged**; **V_resid**
  (run on returns) IS charged ‚Äî cumulative ledger = candidate #1's 6 arms + V_resid's 3.

### Two-gate feature policy (operator-ruled 2026-07-12) ‚Äî vetting ‚âÝ contribution
The stable is built with **two** gates, not one (the IPO advisor's own architecture ‚Äî GMP was in the
model, measured, found not to contribute, and removed):
- **Entry gate ‚Äî standalone vetting.** A feature enters the pool ONLY by showing a **real, independent
  edge alone** (beats random + independent axis). Gap A-Z: **yes** (banked). V_resid: **no** (coin
  flip ‚Üí dead). This is what stops the pool becoming a **feature zoo** ‚Äî you do NOT throw ~250 features
  in and let the model sort them out (that overfits ~2,200 noisy days, and the effective-N-deflated
  DSR bar ‚Äî now 8.97 ‚Äî correctly kills it).
- **Retention gate ‚Äî marginal contribution.** Once in the model, a feature stays only if it
  **measurably improves the held-out metric**, and is cut if it does not (cf. GMP).
- **Vetting gets you into the room; contribution keeps you there.** A **standalone-KILL** (dead to
  trade alone) with a real edge is **BANKED** as a provisional feature, not deleted ‚Äî killing every
  slightly-positive component destroys the ensemble's building blocks (capstone ¬ß2). A **zero-edge**
  result (V_resid) is genuinely dead: nothing to bank. **Small ‚âÝ zero.**
- Stable membership (2026-07-12): **1 provisional member** ‚Äî candidate #1 gap A-Z (banked,
  survivorship-asterisked). Candidate #2 volume-delta: fully dead (V/V-A momentum; V_resid zero edge).
